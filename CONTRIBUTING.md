## Quick Start

1. **Fork** the repository
2. **Clone** your fork: `git clone https://github.com/yourusername/openmindwell.git`
3. **Create a branch**: `git checkout -b feature/your-feature`
4. **Make changes** and test locally
5. **Commit**: `git commit -m "feat: your change description"`
6. **Push**: `git push origin feature/your-feature`
7. **Open a Pull Request** on GitHub

## Development Setup

See [OPENMINDWELL_PROJECT_GUIDE.md](./OPENMINDWELL_PROJECT_GUIDE.md#local-development-setup) for complete setup instructions.

Quick version:
```bash
npm install
cd backend && npm install && cd ..
cd frontend && npm install && cd ..
# Set up .env files (see guide)
npm run dev
```

## What Can I Contribute?

### Good First Issues
- Documentation improvements
- UI/UX enhancements
- Adding mental health resources
- Bug fixes
- Tests

### Ideas
- New guided exercises
- Mobile app (React Native)
- Internationalization (i18n)
- Accessibility improvements
- Advanced analytics

See our [GitHub Issues](https://github.com/yourusername/openmindwell/issues) for current tasks.

## Coding Standards

- **TypeScript**: Use strict mode, avoid `any`
- **React**: Functional components with hooks
- **Code Style**: Follow existing patterns, use ESLint
- **Commits**: Use conventional commits (`feat:`, `fix:`, `docs:`, etc.)
- **Testing**: Test your changes locally before submitting

## Pull Request Process

1. Update documentation if needed
2. Ensure all tests pass (when we add tests)
3. Update OPENMINDWELL_PROJECT_GUIDE.md if adding major features
4. Request review from maintainers
5. Address review feedback

## Code of Conduct

Be kind, respectful, and inclusive. See full Code of Conduct in [OPENMINDWELL_PROJECT_GUIDE.md](./OPENMINDWELL_PROJECT_GUIDE.md#code-of-conduct).
